# RAG-Based Document Question Answering System рҹӨ–рҹ“„

A **Retrieval-Augmented Generation (RAG)** chatbot that lets you upload a PDF
(or .txt file), ask questions about it, and get answers **grounded in the
document's actual content** вҖ” not the model's general training knowledge.

Built entirely on **free tiers**:

| Stage | Tool | Why |
|---|---|---|
| Embeddings | `sentence-transformers` (`all-MiniLM-L6-v2`, 384-dim) | Free, runs locally, no API key needed |
| Vector store | `FAISS` (`IndexFlatIP`, cosine via normalized vectors) | Free, local, no cloud DB account needed |
| Keyword index | `rank-bm25` | Free, enables hybrid search |
| LLM generation | **Groq API** (`llama-3.3-70b-versatile`) | Free tier, very low latency |
| UI | `Streamlit` | Simple local chat interface |

This project demonstrates the same end-to-end pattern as the original
[RAG_Document_Question_Answering](https://github.com/VivekChauhan05/RAG_Document_Question_Answering)
reference repo, with the paid Cohere + Pinecone stack swapped out for a
fully free, locally-run alternative.

## How to Use рҹҡҖ

### 1. Clone / open the project
```bash
cd RAG_Document_QA
```

### 2. Create and activate a virtual environment
```bash
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Add your Groq API key
Get a free key at **https://console.groq.com/keys**, then:
```bash
cp .env.example .env
# edit .env and paste your key into GROQ_API_KEY=
```
> вҡ пёҸ Never commit `.env` вҖ” it's already in `.gitignore`. If a Groq key has
> ever been pasted somewhere public (chat, ticket, repo), regenerate it from
> the Groq console before relying on it.

### 5a. Run the chat app
```bash
cd src
streamlit run app.py
```
Open the URL Streamlit prints (typically `http://localhost:8501`), upload a
PDF in the sidebar, click **Build / Rebuild Index**, and start asking
questions.

### 5b. Or run the CLI validation script
This builds the index from `data/atomic_habits.pdf`, runs a fixed set of
sample questions end-to-end, prints grounded answers to the console, and
writes `docs/validation_log.md`:
```bash
python src/evaluate.py
```

## Key Design Choices

- **Chunking**: sentence-aware sliding window, default `chunk_size=800`
  characters with `chunk_overlap=150` characters, so context isn't cut off
  mid-idea at chunk borders.
- **Embeddings**: `all-MiniLM-L6-v2` (384 dimensions) вҖ” a strong
  speed/quality tradeoff for a fully local, free embedding model.
- **Vector store**: FAISS `IndexFlatIP` over L2-normalized vectors, which is
  mathematically equivalent to cosine similarity search.
- **Hybrid search**: combines min-max-normalized vector similarity and BM25
  keyword scores (`alpha=0.6` weight toward vector search by default),
  configurable in the sidebar or via `Chatbot(search_mode=...)`.
- **Optional re-ranking**: a cross-encoder (`ms-marco-MiniLM-L-6-v2`) can
  re-score a larger shortlist of candidates for higher precision at the cost
  of latency вҖ” toggle with `use_reranker=True`.
- **Grounded generation**: the system prompt instructs the LLM to answer
  *only* from retrieved context and explicitly say so when the answer isn't
  in the document, reducing hallucination.
